Routing Table Entries

The topology displayed in Figure 1 is used as the reference topology for this section. Notice that in the topology:

  • R1 is the edge router that connects to the Internet; therefore, it is propagating a default static route to R2 and R3.

  • R1, R2, and R3 contain discontiguous networks separated by another classful network.

  • R3 is also introducing a 192.168.0.0/16 supernet route.

Figure 2 displays the IPv4 routing table of R1 with directly connected, static, and dynamic routes.

Note: The routing table hierarchy in Cisco IOS was originally implemented with the classful routing scheme. Although the routing table incorporates both classful and classless addressing, the overall structure is still built around this classful scheme.

Figure shows three routers connected linearly through WAN links. R-1 is connected to R-2 through a serial WAN link, and it has a LAN on an Ethernet interface. R-2 is connected to both R-1 and R-3 through serial WAN links, and has a LAN on an Ethernet interface. R-3 is only connected to R-2 through a serial WAN link, and has two LANs on Ethernet interfaces. R-1 has a serial WAN link to the Internet, and router R-3 has a serial WAN link to another cloud network. Figure two shows the output from the command, show i.p. route pipe begin Gateway. This shows the routing table for router R-1, which has three directly connected routes with three interface I.P. addresses, five routes learned through RIP, one candidate default route, and one gateway of last resort configured.
